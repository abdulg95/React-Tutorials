Learning React -  Tutorial 05(React and Ajax)

 

So far, we have been inputting data manually into our state. This has been sufficient in past tutorials but sometimes you might want to populate your app with data from an external api. There are two main ways of doing this with react. The first one is using the in built fetch() function that comes with  react and the other is using the third party library recommended by react called axios. https://github.com/axios/axios . Axios is preferred over fetch because fetch has been known to have a few shortcomings. Axios is used to make http requests from node.js, XMLHttpRequests from browsers and it supports ES6 promises which are objects that represent the eventual completion or failure of an async request . The main feature that it has over fetch() is that it performs automatic transforms of JSON data. In fetch this is split into two where after making an actual request you still need the added step of calling a .json() method to process what is returned from the promise. Here is an example to illustrate what this looks like using both.

 fetch(exampleapiurl).then(response => response.json()).then(data => console.log(data));

In the case above the response is fetched from the url then passed to a json method before it is finally ready to be used.

However using axios this would simply be

axios.get(exampleapiurl).then(response => console.log(response));

 

It may still be preferred to use fetch for small projects, as simply eliminating the middle step is not enough grounds for installing an external library.

Here’s a simple exercise to familiarize yourself with fetching data in the right place in the react component lifecycle. Open the move to the tutorial 5 solution folder and run npm install then npm start. This is a simple react app that fetches data of posts from a site using axios. Observe that the ajax request is made in componentDidMount() part of the lifecycle as recommended in the component lifecycle tutorial. Play around with returned data by slicing it or maybe formatting it into different blocks and make sure you have a firm grasp of what is going on with the code.
